## 1. Автентифікація (AUTH)

| ID | Сценарій | Кроки | Очікуваний результат |
|----|----------|-------|----------------------|
| AUTH-01 | Валідація email | На `/uk/uvijty` ввести `notanemail`, пароль, Submit | Помилка «Некоректний email», запит не йде |
| AUTH-02 | Валідація пароля (min 8) | Ввести валідний email + пароль `123` | Помилка «Мінімум 8 символів» |
| AUTH-03 | Невірні дані | Валідний email, але невірний пароль | Помилка «Невірний email або пароль» |
| AUTH-04 | Логін підтвердженим акаунтом (admin) | Логін admin-кредами | Редірект у `/uk/kabinet`, сесія створена |
| AUTH-05 | Логін непідтвердженим (customer) | Логін customer-кредами | Екран «Підтвердіть свою електронну адресу» + кнопка повторного надсилання |
| AUTH-06 | Повторне надсилання листа | На екрані AUTH-05 натиснути «Надіслати лист повторно» | Стан успіху АБО коректна помилка (sandbox 400) |
| AUTH-07 | Захищений роут без сесії | Відкрити `/uk/kabinet` у новій сесії (logged out) | Редірект на `/uk/uvijty?returnUrl=/uk/kabinet` |
| AUTH-08 | returnUrl після логіну | Залогінитись admin після AUTH-07 | Редірект назад на `/uk/kabinet` |
| AUTH-09 | Гард адмінки (logged out) | Відкрити `/uk/admin` без сесії | Редірект на `/uk/uvijty?returnUrl=/uk/admin` |
| AUTH-10 | Гард адмінки (customer) | Зайти як customer → відкрити `/uk/admin` | Редірект на `/uk/kabinet` (роль не admin/operator) |
| AUTH-11 | Доступ адміна | Зайти як admin → `/uk/admin` | Дашборд відкривається, бейдж ролі ADMIN |
| AUTH-12 | OAuth account_not_linked UX | Відкрити `/uk/uvijty?error=account_not_linked` | Дружній екран «Цей email уже зареєстрований» + поле email + resend (НЕ сирий ERROR) |
| AUTH-13 | Авто-склейка після верифікації | Поставити emailVerified=true → вхід через Google | Better Auth склеює акаунти автоматично (перевірка по коду/БД) |
| AUTH-14 | Вихід | Залогінений → «Вийти» | Сесія завершена, редірект на головну |
| AUTH-15 | Сторінка відновлення паролю | Відкрити `/uk/vidnovyty-parol` | Форма рендериться, submit приймає email |
| AUTH-16 | Реєстрація — валідація | `/uk/reestratsia` з невалідними даними | Помилки валідації |
| AUTH-17 | Реєстрація — успіх | `/uk/reestratsia` з новим email | Екран «перевір пошту» |
| AUTH-18 | Персистентність сесії | Залогінитись → перезавантажити сторінку | Сесія збережена (кука), юзер лишається залогінений |
| AUTH-19 | Бейдж staff на AccountButton | Залогінений admin → іконка акаунта | Значок щита (ShieldCheck) присутній |
